1. Introduction

Welcome to the SNAP Connection Manager. SNAP is a professional-grade GTK3 desktop application built to streamline how system administrators and developers manage SSH and SFTP connections. It combines military-grade encrypted storage, a high-performance VTE terminal emulator, and a fully integrated dual-pane SFTP file manager into a single, cohesive workflow.

2. Initial Setup & Security

SNAP prioritizes the security of your sensitive credentials above all else. Passwords and private keys are never stored in plain text.

Encryption Architecture

  • Master Passphrase: On first launch, you are required to set a Master Passphrase. This phrase is hashed using PBKDF2-HMAC-SHA256 (600,000 iterations) paired with a unique 16-byte cryptographically secure salt.
  • Data Storage: Your server configurations are encrypted symmetrically using AES-256 via GnuPG. The encrypted file is stored locally at ~/.local/share/snap_connection_manager/ssh_servers.json.gpg.
  • Memory Safety: Passwords are decrypted exclusively in memory (RAM) during active sessions and are never written to your disk in plain text.

Changing Your Passphrase

Navigate to File -> Change Passphrase.... You will be prompted for your current passphrase, followed by your new one. SNAP will automatically generate a new salt and securely re-encrypt your entire database.

WARNING: There is no password recovery mechanism. If you forget your Master Passphrase, your data is permanently inaccessible. You will need to manually delete the ~/.local/share/snap_connection_manager/ directory to reset the application.

3. Server & Folder Management

SNAP allows you to logically organize hundreds of servers using the main "Session" tree view.

Adding & Organizing

  • Add a Folder: Right-click the root "Session" folder (or any subfolder) and select Add Folder to create groups (e.g., "Production", "AWS").
  • Add a Server: Right-click a folder and select Add Server. You must provide a Host IP/Domain, Port, Username, and choose either Password or Private Key authentication.
  • Drag-and-Drop: Click and drag servers to move them between folders, or drag folders to rearrange your hierarchy.
  • Natural Sorting: The list automatically sorts items intelligently (e.g., Web-2 is placed before Web-10).

Editing & Actions

  • Inline Renaming: Select a folder or server, wait 1 second, and click it again (or press F2) to rename it directly in the list.
  • Copy/Paste: Right-click a server and select Copy (or press Ctrl + C). Select a destination folder and choose Paste (Ctrl + V) to create an exact duplicate.
  • Deletion: Select an item and press Delete to remove it.

4. Connecting & Terminal Features

4.1 Launching Connections

Double-click any server to instantly open an SSH terminal session. Alternatively, right-click the server and choose either SSH, SFTP (CLI), or SFTP (GUI).

4.2 The VTE Terminal Experience

Connections launch in a native, hardware-accelerated terminal window based on the GNOME VTE library.

  • Smart Copy/Paste: Pressing Ctrl + C will copy text if you have text highlighted. If nothing is highlighted, it intelligently sends a standard interrupt signal (SIGINT) to the server. Use Ctrl + V to paste.
  • Smart Resize: The terminal window size is synchronized with the remote host (SIGWINCH). If you maximize the SNAP window, text-based apps like htop or vim will scale perfectly.
  • Anti-Idle: Keep restrictive firewalls from dropping your connection. Configurable in the server's Terminal tab, SNAP can automatically send hidden "keepalive" null packets at set intervals.

4.3 Live Settings (In-Session)

You do not need to disconnect to change terminal settings. Click the Menu Button (≡) in the header bar of any active terminal window to open the Server Editor. Any changes to Fonts, Colors, or Scrollback are applied instantly when you click OK.

5. SFTP File Manager (GUI)

SNAP includes a powerful, fully graphical SFTP client for visual file management. Right-click any server and select SFTP (GUI) to open it.

5.1 Interface & Navigation

  • Dual-Pane View: Your Local Machine is displayed on the left, and the Remote Server is on the right.
  • Breadcrumb Navigation: Click any part of the folder path at the top of a pane to jump directly to that directory.
  • Refresh: Use the global refresh button (circular arrows) in the top toolbar to reload both panes simultaneously.

5.2 Smart Transfers

  • Drag and Drop: Click and drag files or folders from one pane to the other to initiate an upload or download.
  • Toolbar Transfers: Select items and click the Green (Upload) or Blue (Download) transfer buttons in the toolbar.
  • Resume Transfers: If a large transfer is interrupted, select the file again and click the Yellow Fast Forward button. SNAP will calculate the size difference and only transfer the missing bytes.
  • Cancel: Click the Red Stop button to abort any ongoing transfer or deep search.

5.3 File Management & Safe Deletion

  • Context Menu: Right-click empty space or files to Create Directories, Rename (F2), Copy (Ctrl + C), or Cut (Ctrl + X).
  • Safe Trash (Local): Pressing Delete on a local file safely moves it to your operating system's Rubbish Bin.
  • Permanent Delete: Press Shift + Delete to permanently destroy a local file.
    Note: All deletions performed on the Remote Server are always permanent.

5.4 Recursive Search

Find files buried deep within large folder structures without manually clicking through them.

  • How to Search: Click the Binoculars icon in the toolbar to search the currently active pane, or right-click a specific folder and choose Search Here...
  • Results: A window will appear with matching files. Double-click any result to instantly navigate the active pane to that exact file's location.

6. Advanced Automation & Tunneling

6.1 Startup Command Files

Automate your workflow by having SNAP execute commands immediately after a successful login.

  1. Create a local text file (e.g., startup.txt) containing the commands you want to run (one per line).
  2. In the Server Editor, navigate to the Terminal tab.
  3. Select your text file in the "Startup Command File" field.

Common Use Cases: Sourcing a virtual environment, navigating directly to a deep project directory (cd /var/www/html), or launching a monitoring tool (htop) automatically upon connect.

6.2 Login Actions (Expect/Send)

Handle complex authentication hurdles like secondary passwords, 2FA prompts, or legal disclaimers.

  1. In the Server Editor, go to the Login Actions tab and click Add Row.
  2. Expect: Enter the exact text the server outputs (e.g., Verification code:).
  3. Send: Enter the response SNAP should type.
  4. Hide: Check this box if the "Send" text is a sensitive password. It will be masked in the UI and stripped from session logs.

6.3 Port Forwarding

Tunnel traffic securely through your SSH connection via the Port Forwarding tab.

  • Local (-L): Access a service on the remote network via your local machine (e.g., routing `8080` locally to an internal database).
  • Remote (-R): Expose a local service on your machine to the remote server.
  • Dynamic (-D): Creates a local SOCKS5 proxy that allows you to route standard browser traffic through the remote server.

7. Logging & Data Management

7.1 Clean Session Logging

SNAP features a sophisticated logging engine perfect for troubleshooting and auditing.

  • Enable Logging: In the Terminal tab, check "Enable Logging" and select an output file path.
  • Append vs. Overwrite: Choose whether to add new session data to the end of an existing file (historical tracking) or wipe it clean every time you connect (current state debugging).
  • "Clean Text" Technology: SNAP automatically strips out terminal control codes (ANSI colors, window title updates) in real-time, resulting in clean, human-readable text perfect for use with grep.

7.2 Import / Export

Backup or share your server configurations via the File menu.

  • JSON Export: Saves a plaintext .json file. Warning: Passwords are exposed in plain text.
  • GPG Export: Saves an encrypted .json.gpg file. You will be prompted for your Master Passphrase to encrypt it, allowing you to safely move your configuration between computers.

8. Appearance & Customization

8.1 Global Defaults

Navigate to File -> Default Settings... to set your baseline preferences. You can choose a default Font, Text Color, Background Color, and built-in Palette (e.g., Solarized, Tango, GNOME). Any newly created server will automatically inherit these settings.

8.2 Per-Server Customization

You can override global settings on a per-server basis in the Appearance tab of the Server Editor. If you ever want to revert a highly customized server back to standard, click the Reset to Defaults button.

9. Installation & Troubleshooting

Quick Install Script

curl -sL https://raw.githubusercontent.com/larreblarsson/SNAP-Connection-Manager/main/install.sh | bash

Manual APT Installation

Run the following commands to add the official repository and install the application:

# Add the GPG key
curl -sL https://larreblarsson.github.io/SNAP-Connection-Manager/public.key | sudo gpg --dearmor -o /usr/share/keyrings/SNAP-Connection-Manager-keyring.gpg

# Add the repository (64-bit architecture only)
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/SNAP-Connection-Manager-keyring.gpg] https://larreblarsson.github.io/SNAP-Connection-Manager stable main" | sudo tee /etc/apt/sources.list.d/SNAP-Connection-Manager.list

# Update and install dependencies/app
sudo apt update
sudo apt install snap-connection-manager
